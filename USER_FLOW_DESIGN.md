## 전체 사용자 여정 개요

### 사용자 생애주기 (User Lifecycle)

```
1. 인식 (Awareness)
   └─ 마케팅/검색 → 랜딩 페이지

2. 획득 (Acquisition)
   └─ 첫 방문 → 회원가입 → 온보딩

3. 활성화 (Activation)
   └─ 첫 기능 사용 → 가치 경험 → 재방문

4. 유지 (Retention)
   └─ 정기적 사용 → 습관 형성 → 충성도 증가

5. 수익화 (Revenue)
   └─ 프리미엄 전환 → 구독 → 결제

6. 추천 (Referral)
   └─ 만족도 증가 → 지인 추천 → 바이럴
```

## 주요 페르소나

### 페르소나 1: 직장인 김민수 (30대 남성)
**목표**: 출퇴근 시간 단축, 효율적인 주차 관리  
**Pain Point**: 매일 주차장 찾기, 교통 체증  
**사용 패턴**: 주중 아침/저녁, 주말 가족 외출  
**주요 기능**: 주차장 예약, 경로 찾기, 실시간 교통 정보

### 페르소나 2: 여행자 이지은 (20대 여성)
**목표**: 새로운 장소 탐험, 여행 계획 관리  
**Pain Point**: 정보 과부하, 일정 관리 어려움  
**사용 패턴**: 주말/휴가 기간 집중 사용  
**주요 기능**: 여행 플래너, 목적지 추천, 예약 관리

### 페르소나 3: 소상공인 박상호 (40대 남성)
**목표**: 매장 주변 고객 유치, 불편사항 신속 해결  
**Pain Point**: 주차 문제로 고객 이탈, 시설 문제  
**사용 패턴**: 매일 사용, 민원 신고 빈번  
**주요 기능**: 주변 안내, 민원 신고, 주차 정보

## 핵심 User Flow

### Flow 1: 첫 방문자 → 활성 사용자

```
┌─────────────────────────────────────────────────────────────┐
│  1. 랜딩 (Landing)                                          │
├─────────────────────────────────────────────────────────────┤
│  • 메인 페이지 도착                                         │
│  • 주요 기능 한눈에 파악 (히어로 섹션)                      │
│  • 명확한 가치 제안 (Value Proposition)                     │
│  └─→ [시작하기] 또는 [둘러보기]                           │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 온보딩 (Onboarding) - 3단계 5분 이내                    │
├─────────────────────────────────────────────────────────────┤
│  Step 1: 환영 및 소개                                       │
│    • "안녕하세요! 마이맵봇입니다 👋"                        │
│    • 핵심 기능 3가지 소개 (슬라이드)                        │
│                                                              │
│  Step 2: 회원가입 (선택적)                                  │
│    • 간편 가입 (소셜 로그인 지원)                           │
│    • 또는 [나중에 하기]로 체험 먼저                         │
│                                                              │
│  Step 3: 개인화 설정                                        │
│    • 주요 관심사 선택 (주차/여행/식당 등)                   │
│    • 선호 교통수단 선택                                     │
│    • 알림 설정 (선택적)                                     │
│  └─→ [시작하기]                                            │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 첫 기능 사용 (First Value)                              │
├─────────────────────────────────────────────────────────────┤
│  • 가이드 툴팁 표시 (첫 3회 사용)                           │
│  • "현재 위치 주변 주차장을 찾아볼까요?"                    │
│  • 실시간 결과 표시 → 즉각적인 가치 제공                   │
│  • 성공 축하 메시지 + 다음 단계 제안                        │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 습관 형성 (Habit Building)                              │
├─────────────────────────────────────────────────────────────┤
│  • 자주 찾는 장소 자동 학습                                 │
│  • 사용 패턴 분석 → 맞춤 추천                               │
│  • 주간 리포트 (사용 통계, 절약 시간 등)                    │
│  • 리워드 시스템 (포인트, 배지)                             │
└─────────────────────────────────────────────────────────────┘
```

### Flow 2: 주차장 찾기 (핵심 기능)

```
사용자 의도: "지금 주차할 곳을 찾고 싶다"

1. 진입점 (Entry Point)
   ├─ 메인 대시보드 [주차] 탭
   ├─ 챗봇 "주차장 찾기"
   ├─ 음성 명령 "근처 주차장"
   └─ 푸시 알림 (자주 가는 장소 근처)

2. 검색/필터
   ├─ 자동: 현재 위치 기반
   ├─ 수동: 목적지 주소 입력
   └─ 필터:
       ├─ 빈 자리만
       ├─ 요금 범위
       ├─ 거리
       └─ 시설 (실내/옥외, EV충전)

3. 결과 표시
   ├─ 지도에 마커
   ├─ 리스트 (거리순/요금순)
   └─ 각 주차장 정보:
       ├─ 실시간 빈 자리
       ├─ 요금 (시간당)
       ├─ 거리 (도보 시간)
       ├─ 평점 및 리뷰
       └─ 영업 시간

4. 액션
   ├─ [예약하기] → 예약 확인 → 알림 설정
   ├─ [경로 안내] → 네비게이션 시작
   ├─ [즐겨찾기] → 내 즐겨찾기에 추가
   └─ [공유하기] → 링크 공유

5. 후속 조치
   ├─ 예약 알림 (도착 30분 전)
   ├─ 주차 시간 기록
   ├─ 출차 알림 (사용자 설정 시간)
   └─ 리뷰 요청 (출차 후)
```

### Flow 3: 여행 계획 (복합 기능)

```
사용자 의도: "주말 여행을 계획하고 싶다"

1. 시작
   ├─ [여행] 탭 → [플래너]
   └─ 챗봇: "여행 계획을 도와드릴게요!"

2. 기본 정보 입력
   ├─ 목적지 (또는 추천받기)
   ├─ 날짜 및 기간
   ├─ 인원
   ├─ 예산 (선택)
   └─ 여행 스타일 (관광/휴양/미식 등)

3. AI 추천
   ├─ 개인화된 목적지 추천
   │   ├─ 과거 여행 이력 분석
   │   ├─ 선호도 기반
   │   └─ 시즌/날씨 고려
   ├─ 숙박 추천
   ├─ 식당 추천
   └─ 관광지 추천

4. 일정 구성
   ├─ 자동 일정 생성 (AI)
   ├─ 수동 편집 (드래그앤드롭)
   ├─ 시간별 배치
   └─ 경로 최적화

5. 예약 통합
   ├─ 숙박 예약
   ├─ 식당 예약
   ├─ 주차장 예약
   └─ 액티비티 예약

6. 여행 진행
   ├─ 당일 일정 알림
   ├─ 실시간 네비게이션
   ├─ 변경사항 즉시 반영
   └─ 현지 추천 (근처 맛집 등)

7. 여행 후
   ├─ 추억 저장 (사진, 메모)
   ├─ 리뷰 작성
   ├─ 통계 (방문 장소, 비용 등)
   └─ 다음 여행 추천
```

## 기능별 상세 Flow

### 1. 회원가입 및 온보딩

#### A. 소셜 로그인 (간편 가입)
```
[구글/카카오/네이버 로그인] 클릭
  ↓
소셜 인증
  ↓
계정 연동
  ↓
기본 정보 자동 입력
  ↓
추가 정보 입력 (선택적)
  - 전화번호
  - 생년월일
  ↓
이용약관 동의
  - 필수: 서비스 이용약관, 개인정보처리방침
  - 선택: 마케팅 수신 동의, 위치정보 이용 동의
  ↓
회원가입 완료
```

#### B. 온보딩 튜토리얼
```
Step 1: 환영 화면
  "안녕하세요! 마이맵봇에 오신 것을 환영합니다 🎉"
  [시작하기] [둘러보기]

Step 2: 핵심 기능 소개 (3개 슬라이드)
  1. "주차장 찾기 - 실시간 빈 자리 확인 🅿️"
  2. "여행 플래너 - AI가 추천하는 맞춤 일정 ✈️"
  3. "주변 탐색 - 내 주변 모든 것 🔍"
  
  [이전] [다음] [건너뛰기]

Step 3: 개인화 설정
  "어떤 서비스에 관심이 있으신가요?"
  
  □ 주차장 찾기
  □ 경로 안내
  □ 여행 계획
  □ 맛집 탐색
  □ 주변 시설 안내
  
  "주로 어떤 교통수단을 이용하시나요?"
  ○ 자동차  ○ 대중교통  ○ 도보  ○ 자전거
  
  [완료]

Step 4: 첫 액션 제안
  "🎯 지금 바로 시작해볼까요?"
  
  - [내 주변 주차장 보기]
  - [여행 계획 시작하기]
  - [주변 맛집 찾기]
  
  또는 [나중에 하기]
```

### 3. 개인화 추천 엔진

#### 추천 알고리즘 전략
```
1. 협업 필터링 (Collaborative Filtering)
   - 유사한 사용자의 선호도 분석
   - "이 장소를 방문한 사람들은 ..."

2. 콘텐츠 기반 필터링 (Content-Based)
   - 사용자 과거 선택 기반
   - 유사한 특성의 장소 추천

3. 하이브리드 접근
   - 협업 + 콘텐츠 기반 결합
   - 가중치 조정으로 최적화

4. 컨텍스트 인식 (Context-Aware)
   - 시간: 아침/점심/저녁
   - 날씨: 맑음/비
   - 위치: 현재 위치
   - 요일: 주중/주말
   - 동반자: 혼자/가족/친구
```

#### 추천 시나리오 예시

**시나리오 1: 점심 식당 추천**
```
사용자: 김민수 (직장인, 30대)
시간: 12:00 (점심)
위치: 회사 근처 (강남역)
날씨: 맑음
요일: 수요일

AI 추론:
1. 과거 데이터 분석
   - 점심: 주로 한식 (70%), 빠른 식사 선호
   - 예산: 평균 10,000원
   - 거리: 도보 10분 이내

2. 컨텍스트 고려
   - 점심 시간대 → 빠른 서빙 필수
   - 회사 근처 → 도보 거리
   - 수요일 → 특별 할인 있는 식당 우선

3. 추천 결과 (Top 3)
   ① 참숯갈비 (한식, 9,000원, 도보 5분, 평점 4.5)
      이유: 자주 방문, 빠른 서빙, 점심 특선
   
   ② 이탈리아 파스타 (양식, 11,000원, 도보 7분, 평점 4.7)
      이유: 새로운 경험, 좋은 리뷰, 분위기 좋음
   
   ③ 일본식 우동 (일식, 8,500원, 도보 3분, 평점 4.3)
      이유: 빠른 식사, 저렴한 가격, 가까운 거리
```

**시나리오 2: 주말 여행지 추천**
```

   - 숙소: 감성 펜션 선호

2. 시즌 고려
   - 봄꽃 명소 추천
   - 날씨 좋은 야외 활동
   - 축제 정보

3. 추천 일정
   목적지: 강릉 (바다 + 카페 + 봄꽃)

   12:00 정동진 해변 (점심)
   14:00 순두부 마을 (카페 투어)

   Day 2:
   08:00 일출 보기
   10:00 안목 카페 거리
   12:00 점심 (해물 칼국수)
   14:00 귀가
   
   예상 비용: 285,000원
   포인트 할인 가능: -15,000원
```

### 4. 히스토리 및 즐겨찾기

#### 히스토리 관리
```
1. 자동 기록
   - 검색 내역 (최근 100개)
   - 방문 장소 (자동 체크인)
   - 예약 내역 (전체 보관)
   - 경로 이력

2. 수동 관리
   - [삭제] 개별 또는 전체
   - [비공개] 특정 항목 숨김
   - [내보내기] CSV/PDF

3. 분석 대시보드
   - 주간/월간 통계
   - 자주 가는 장소 Top 10
   - 총 이동 거리
   - 절약한 시간
   - 사용한 포인트
```

#### 즐겨찾기 시스템
```
1. 폴더 관리
   - 기본 폴더: 주차장, 식당, 카페, 여행지
   - 커스텀 폴더: 사용자 생성
   - 공유 폴더: 친구와 공유

2. 스마트 리스트
   - 자주 가는 곳 (AI 자동 분류)
   - 주중 vs 주말
   - 시간대별
   - 카테고리별

3. 빠른 액션
   - 원탭 네비게이션
   - 즐겨찾기 위젯 (홈화면)
   - 음성 명령 지원
```

### 5. 알림 시스템

#### 알림 타입 및 우선순위

**1순위: 예약 관련 (Critical)**
```
- 예약 확인
- 도착 시간 알림 (30분 전)
- 출차 알림
- 예약 취소 알림
```

**2순위: 개인화 추천 (High)**
```
- 출근 시간 주차장 추천
- 점심 시간 식당 추천
- 주말 여행지 추천
```

**3순위: 정보성 (Medium)**
```
- 주간 리포트
- 포인트 적립 알림
- 새 기능 안내
```

**4순위: 마케팅 (Low - 선택적)**
```
- 프로모션
- 이벤트
- 뉴스레터
```

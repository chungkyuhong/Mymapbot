## 🔧 수정된 문제점

### 1. 여행 탭 UI 문제
**문제**: 복잡한 HTML 구조로 인한 혼란
**해결**: 단순화된 구조로 재설계
- 불필요한 입력 필드 제거 (목적지, 날짜, 인원)
- 여러 개별 컨테이너를 단일 `recommendationList`로 통합
- 쇼핑/문화 버튼 제거 (API 미지원)

### 2. JavaScript 이벤트 버그
**문제**: `selectPurpose` 함수에서 `event.target` 참조 오류
**해결**: 버튼 찾기 로직 개선
```javascript
// Before
event.target.classList.add('active');

// After
document.querySelectorAll('.purpose-btn').forEach(btn => {
    if (btn.getAttribute('data-purpose') === purpose) {
        btn.classList.add('active');
    }
});
```

- `navigator.geolocation.getCurrentPosition()` 활용

- 위치 권한 요청 처리


**작동 방식**:
1. 사용자가 "📍 현재 위치" 버튼 클릭
2. 브라우저가 위치 권한 요청
3. 허용 시: GPS 좌표 획득 → 지도 이동
4. 거부 시: 기본 위치(서울시청) 사용

### ✅ 2. 여행 추천 링크
```
여행 탭 클릭
→ 목적 선택 (출장/여행/식사)
→ API 호출
→ 추천 정보 표시
→ 지도 마커 표시
```

**API 엔드포인트**:
- `GET /api/recommendations/business` - 출장 추천
- `GET /api/recommendations/travel` - 여행 추천
- `GET /api/recommendations/dining` - 식사 추천

**파라미터**:
- `lat`: 현재 위도
- `lng`: 현재 경도
